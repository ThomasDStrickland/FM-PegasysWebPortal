Pegasys Fixed Assets Phase 2 Training for PBS Contents

Introduction
This training segment provides an overview of the Pegasys Fixed Assets Phase 2 subsystem. It covers key concepts that are critical to understanding the demonstrations, such as the fixed asset lifecycle and referencing fixed asset records on purchasing documents.

Fixed Asset Purchase Order Demonstrations

Demonstration 1: Purchase Order with One Header Accounting Line to One Asset Record.
This demonstration covers a construction purchase order (PN) referencing one asset record for the full amount of the purchase order.

Demonstration 2: Purchase Order with One Header Accounting Line to Multiple Asset Records.
This demonstration covers a non-recurring purchase order (PJ) referencing two different personal property asset records.

Demonstration 3: Purchase Order with Multiple Header Accounting Lines to Multiple Asset Records.
This demonstration covers a non-recurring purchase order (PJ) with two different R-type projects that also reference two different asset records.

Demonstration 4: Purchase Order with Multiple Header Accounting Lines to one Asset Record.
This demonstration covers a construction purchase order (PN) which has two different accounting lines (FC: PG413 and FC: PG511) but reference only one asset record.

Fixed Asset Expenditure Document Demonstrations

Demonstration 5: Receiving Report
This demonstration covers a simple construction purchase order (PN) partial receiving report (RW) and linking it to the asset record. It provides a step-by-step guide of linking the asset record and changing the asset accounting line for partial receipt amount.

Demonstration 6: Estimated Accrual
This demonstration covers a simple non-recurring purchase order (PJ) creation of estimate accrual (EA) and linking it to the asset record. It provides a step-by-step guide of linking the asset record and changing the asset accounting line for partial accrual amount.

Demonstration 7: Credit Card Log Reconciliation
This demonstration covers a simple credit card reconciliation creation (CL) and linking to the asset record. It provides a step-by-step guide of linking the asset record and changing the asset accounting line for reconciliation purposes.

Fixed Asset Queries

Demonstration 8: Pegasys Fixed Asset Query
The Asset Query contains a complete set of information available for a given asset and its betterments. It reflects the current status of the asset and provides a history of the activity processed against the asset record. This query is used to research information regarding the asset or betterment.

Demonstration 9: Pegasys Fixed Assets Journal Query
The purpose of the Fixed Assets Journal is to provide an audit trail of all asset-related activity. It supplements the Transaction, Accounting, and General Journals by serving as a subsidiary ledger, and tracks asset-related transactions and asset-specific information that are not included in the other journals.

Demonstration 10: Pegasys General Ledger Detail Query
The GL Account Detail query is used as an alternate way of viewing information contained in the ledger. Over 40 pieces of data to narrow the query results to only those records the user wants to view. The results of the query are displayed in a tabular format where each posting (a single debit or credit) recorded for each document line is displayed.
